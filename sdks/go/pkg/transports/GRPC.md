# gRPC Transport

`GrpcTransport` exposes Catena handlers through the generated `CatenaService` gRPC API.

## Constructor

```go
grpcTransport := transports.NewGrpcTransport(config.GrpcOptions{Port: 6254, Reflection: false})
// or with defaults
grpcTransport := transports.NewGrpcTransport(config.DefaultGrpcOptions())
```

Register it on the shared server:

```go
srv := catena.NewServer(100)
if err := srv.RegisterTransport(grpcTransport); err != nil {
    panic(err)
}
```

## Implemented RPCs

Implemented against shared runtime handlers:

- `GetPopulatedSlots`
- `DeviceRequest`
- `GetValue`
- `GetParam`
- `SetValue`
- `MultiSetValue`
- `ExternalObjectRequest`
- `ExecuteCommand`
- `ParamInfoRequest`
- `Connect`

Currently unimplemented (returns `Unimplemented`):

- `UpdateSubscriptions`
- `AddLanguage`
- `LanguagePackRequest`
- `ListLanguages`
- `RefreshToken`
- `RevokeAccess`

## Handler Mapping

RPC methods invoke the same handlers registered on `catena.Server`:

- `DeviceRequest` -> `RegisterGetDeviceHandler`
- `GetValue` -> `RegisterGetValueHandler`
- `GetParam` -> `RegisterGetParamHandler`
- `SetValue` / `MultiSetValue` -> `RegisterSetValueHandler` (the handler receives `[]SetValueEntry`; `SetValue` delivers a one-element slice, `MultiSetValue` delivers the full slice for atomic application)
- `ExternalObjectRequest` -> `RegisterGetAssetHandler`
- `ExecuteCommand` -> `RegisterExecuteCommandHandler`
- `ParamInfoRequest` -> `RegisterParamInfoHandler`

## Streaming Updates (`Connect`)

`Connect` registers a transport-owned connection and streams `PushUpdates` generated by:

```go
srv.BroadcastUpdate(slot, "volume", 42)
```

Every active push stream across transports receives the update through shared server connection management.

Connection lifecycle details:

- When a client calls `Connect`, the transport registers with `RegisterTransportConnection(owner)`.
- The returned `connection.Updates` channel is the source of push updates to forward on the gRPC stream.
- The returned `connection.Done` channel signals server-side shutdown/cleanup so the stream can exit gracefully.
- The transport calls `DeregisterConnection(connID)` when the stream ends.

Why owner is required:

- Connections are associated with the transport instance that created them.
- This enables targeted cleanup through `ShutdownTransportConnections(owner)`.
- A single transport can deregister/shutdown and clean up only its own connections without terminating streams owned by other transports.

## Error Mapping

Catena status codes are translated with `ToGRPCCode`.

Examples:

- `OK` -> `codes.OK`
- `NOT_FOUND` -> `codes.NotFound`
- `INVALID_ARGUMENT` -> `codes.InvalidArgument`
- `UNIMPLEMENTED` -> `codes.Unimplemented`
- `UNAVAILABLE` -> `codes.Unavailable`

In production mode, detailed error text is sanitized to status code names.

## Reflection

Reflection can be enabled at construction time:

```go
grpcTransport := transports.NewGrpcTransport(config.GrpcOptions{Port: 6254, Reflection: true})
```

When enabled, tools such as `grpcurl` can discover services and methods dynamically.

## Graceful Shutdown

`GrpcTransport.Shutdown(ctx)` performs graceful stop first, then forces stop if context expires.

Because `grpc.Server.GracefulStop()` waits for active streams to finish, gRPC shutdown should also request owner-scoped runtime stream signaling:

- gRPC starts graceful stop so no new RPCs are accepted.
- gRPC requests `ShutdownTransportConnections(ctx, owner)` so active `Connect` streams receive `connection.Done` and exit.
- gRPC waits for graceful completion and falls back to forced stop on context timeout.
- Server-level shutdown still provides a final force-drain safety net.

Context contract:

- `Start(ctx, runtime)` accepts `ctx` for future-proof transport lifetime control, even when current startup paths do not fully depend on it.
- `Shutdown(ctx)` must respect `ctx` as a deadline/cancel signal and MUST return by that deadline.

Caller guidance:

- Prefer `context.WithTimeout` when calling shutdown (`srv.Shutdown(ctx)` or `srv.DeregisterTransport(ctx, grpcTransport)`).
- gRPC graceful shutdown can hang without a deadline; always providing one makes shutdown behavior predictable.

At the server level:

```go
srv.Shutdown(ctx)
srv.Wait()
```

To remove only gRPC transport:

```go
_ = srv.DeregisterTransport(ctx, grpcTransport)
```
